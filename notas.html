CONCEPTOS GENERALES

Estos conceptos se basan en los modelos de diseño de la web a lo largo de la historia.

TIPOS DE WEB

  • WEB 1.0: La web estática, solo permitía la lectura de información sin interacción dinámica.
  • WEB 1.5 (Dinámica): Comienza a introducirse la interactividad, donde los usuarios pueden realizar acciones limitadas como formularios, pero aún no hay tanta integración.
  • WEB 2.0 (Social): La web interactiva, con plataformas de redes sociales, creación de contenido y participación activa de los usuarios. Ejemplos incluyen plataformas como Facebook, Twitter, y YouTube.
  • WEB SEMÁNTICA: La web avanzada que integra inteligencia artificial (IA). Permite a las máquinas procesar y entender el contenido de manera más profunda, facilitando la interacción natural entre humanos y máquinas mediante tecnologías como el Procesamiento Natural del Lenguaje (PNL).

Los métodos HTTP más utilizados son GET y POST.

ALGUNOS CONCEPTOS CLAVE

  • URL (Uniform Resource Locator): Dirección utilizada para localizar recursos en la web, como páginas HTML, imágenes, y videos. Permite acceder a sitios web y comunicarse a través de Internet.
  • HTTP (HyperText Transfer Protocol): Protocolo utilizado para la transmisión de datos entre el navegador y el servidor web. Se encarga de definir cómo los mensajes son enviados y recibidos.
  • HTML (HyperText Markup Language): Lenguaje estándar para la creación y estructura de páginas web. Usa etiquetas para definir elementos como encabezados, párrafos, imágenes y enlaces.
  • XHTML (Extensible HyperText Markup Language): Versión más estricta de HTML, basada en XML. Mejora la interoperabilidad y garantiza una sintaxis más rigurosa en los documentos.
  • CSS (Cascading Style Sheets): Lenguaje que se usa para describir el diseño visual de los documentos HTML o XML. Permite separar el contenido (HTML) de la presentación (estilos visuales).
  • RIA (Rich Internet Applications): Aplicaciones web que funcionan como programas de escritorio, pero se ejecutan en un navegador. Proveen una experiencia interactiva avanzada, usando tecnologías como AJAX y Flash.
  • SOAP (Simple Object Access Protocol): Protocolo basado en XML para intercambiar información estructurada entre aplicaciones, comúnmente utilizado en servicios web. (mas completo que REST)
  • REST (Representational State Transfer): Estilo arquitectónico para diseñar servicios web. A diferencia de SOAP, utiliza los métodos estándar de HTTP (GET, POST, PUT, DELETE) y se enfoca en la simplicidad y la interoperabilidad.
  • XML (Extensible Markup Language): Lenguaje de marcado que permite codificar documentos de manera estructurada. Es utilizado para almacenar y transportar datos de manera legible tanto para humanos como para máquinas.
  • UDDI (Universal Description, Discovery, and Integration): Estándar que permite registrar y descubrir servicios web. Facilita el descubrimiento de servicios y su integración.
  • WSDL (Web Services Description Language): Lenguaje basado en XML que describe los servicios web, sus funciones y los parámetros que aceptan.
  • WS (Web Services): Son servicios que se comunican a través de la web, permitiendo la interoperabilidad entre aplicaciones y plataformas. Usan protocolos como SOAP o REST para la comunicación.
  • Navegador Web: Software que permite acceder a la información en la web. Los navegadores interpretan el código HTML y lo presentan de forma visual.
  • Social Media (Redes Sociales): Plataformas online donde los usuarios pueden crear contenido, interactuar con otros y compartir información en tiempo real. Ejemplos son Facebook, Twitter e Instagram.
  • Identidad Digital: Representación online de una persona o entidad, creada a través de interacciones en redes sociales y otros servicios en línea. Permite la autenticación y el acceso a plataformas digitales.

HTTP(S)

HTTP es el protocolo fundamental para la comunicación en la web, pero con la creciente necesidad de seguridad, se implementa su versión segura, HTTPS, que cifra la información para protegerla de ataques como la interceptación de datos (Man-in-the-middle). DOM (Document Object Model)

El DOM es una interfaz que permite a los programadores interactuar con los documentos HTML y XML. Representa la estructura del documento como un árbol de nodos, donde cada parte del documento (como elementos, atributos o texto) es un nodo que puede ser manipulado. Con JavaScript, puedes modificar dinámicamente los contenidos y la apariencia de una página web sin necesidad de recargarla, mejorando la interactividad.
